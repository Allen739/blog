# How to Create Interactive Blog Posts with MDX

This blog is set up so you can publish advanced, interactive posts by editing only files in `content/*.mdx`.
You do not need to touch React page code for normal publishing.

## What You Can Deliver with `.mdx`

With MDX in this repo, a single post can include:

- rich prose and structured writing
- syntax-highlighted code
- callouts and pull quotes
- animated stats
- before/after image scrubbers
- parallax media sections
- full-bleed hero blocks
- embedded video
- custom stat dashboards

## 1. Start with File + Frontmatter

Create a file:

`content/your-post-slug.mdx`

Use this frontmatter:

```mdx
---
title: "Your Post Title"
publishedAt: "2026-02-16"
summary: "One clear sentence about the value of this post."
layout: "standard"
---
```

## 2. Write Strong Core Content First

Use normal Markdown/MDX for the narrative:

```mdx
## The Problem

Explain what is broken or painful.

## The Approach

Explain your reasoning and tradeoffs.

## The Result

Show outputs, data, and what changed.
```

## 3. Use Rich Building Blocks

### A. Highlighted Code Blocks

````mdx
```ts
export async function shipFeature() {
  const result = await deploy();
  return result.status;
}
```
````

### B. Annotated Code (line guidance)

````mdx
```ts {2: Call the API}{4: Handle network failure}
const response = await fetch("/api/run");
const data = await response.json();
if (!response.ok) {
  throw new Error("Request failed");
}
```
````

### C. Callouts

```mdx
<Callout type="info">
Use this when you want to add context without breaking flow.
</Callout>
```

Supported `type` values:
- `default`
- `info`
- `success`
- `warning`
- `error`

### D. Pull Quotes

```mdx
<PullQuote attribution="From my deployment notes">
The biggest speed boost came from reducing feedback loops, not from writing faster code.
</PullQuote>
```

## 4. Image and Asset Rules

- Put images/videos in `public/` (example: `public/blog/my-image.png`).
- Refer to them in MDX as `/blog/my-image.png`.
- If using external images with `RoundedImage`, the host must be allowed in `next.config.mjs`.

## 5. Writing Style That Works Well

- Start with a clear promise in the first 3 lines.
- Keep sections short and scannable.
- Add one concrete example per major claim.
- Prefer data over adjectives.
- End with a reusable checklist or template.

## 7. Publish in 4 Commands

```bash
git add content/your-post-slug.mdx
git commit -m "Add blog post: your short title"
git push
```

That is enough for Vercel auto-deploy.

## 8. Common Failure Modes

- Missing `---` in frontmatter
- Invalid `publishedAt` format
- Bad MDX syntax (unclosed tags/code fences)
- Wrong media path (file not in `public/`)
- Using a custom component name that is misspelled

## 9. Quality Checklist Before Push

Run this mental checklist:

- Does the title promise specific value?
- Is the summary clear on the blog listing page?
- Is every section adding new information?
- Are visuals tied to a specific insight?
- Did you include one practical takeaway?

If yes, ship it.
